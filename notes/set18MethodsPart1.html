Skill 18.2 Write code to create and implement a method
Skill 18.2 Concepts

In our previous example, every time the user wanted to make a deposit, make a withdrawel, or check their balance we had to add code to the main method of our program. Can you imagine how messy it would look if you needed to make 10 deposits, followed by 10 more balance inquiries?

Methods in java can be used to define such behaviors and can be applied to greatly simplify our code. Consider checkBalance method below.

public void checkBalance(){

System.out.println("Your balance is " + balance);

}

The first line, public void checkBalance(), is the method signature. It gives the program some information about the method. each part of this line is described below,

  • public means that other classes can access this method. We will learn more about that later.
  • The void keyword means that there is no specific output from the method. We will see methods that are not void later, but for now all of our methods will be void.
  • checkBalance is the name of the method.

The statement System.out.println("Your balance is " + balance); is in the body of the method, which is defined by the curly braces. Anything we can do in our main() method, we can do in other methods! All of the Java tools you know, like the math and comparison operators, can be combined to create useful methods.

Now that we have added the checkBalance() method to our SavingsAccount class, it is available to use on any SavingsAccount object. We can do this by calling the method on the SavingsCount object we create, for example.

In the example above, we call the checkBalance() method on the account1 object. This method call occurs inside the main() method. Running the program results in printing Your balance is $125.00 to the console.

Skill 18.3 Identify the scope of a variable
Skill 18.3 Concepts

A method is a task that an object of a class can perform. We mark the domain of this task using curly braces: {, and }. Everything inside the curly braces is part of the task. This domain is called the scope of a method.

We cannot access variables that are declared inside a method in code that is outside the scope of that method.

Consider the SavingsAccount class below,

public class SavingsAccount{

public String name;
public double balance;

public SavingsAccount(String n, double b){

name = n
balance = b
}
public void checkBalance(){

String message = "Your balance is $" + balance;
System.out.println(message);
}
}

The variable message, which is declared and initialized inside of checkBalance(), cannot be used inside any other method. It only exists within the scope of the checkBalance() method. However, name and balance, which are declared at the top of the class, can be used inside all methods in the class, since they are in the scope of the whole class.

Similar to how we added parameters to constructors, we can customize all other methods to accept parameters.

public class SavingsAccount{

public String name;
public double balance;

public SavingsAccount(String n, double b){

balance = b
}
public void checkBalance(){

String message = "Your balance is $" + balance;

}
public void depositMoney(double d){

balance += d;
}
}

In the code above, we created a depositMoney() method that accepts an double parameter called d. In the main() method, we can call the depositMoney() method on an object and provide a double parameter to increase the balance.
